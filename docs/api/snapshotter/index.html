Snapshotter

Creates snapshots of system resources at given intervals and marks the resource as either overloaded or not during the last interval. Keeps a history of the snapshots. It tracks the following resources: Memory, EventLoop and CPU. The class is used by the AutoscaledPool class.

There are differences in behavior when running locally and on the Apify platform, but those differences are handled internally by the class and do not affect its interface.

Memory becomes overloaded if its current use exceeds the maxUsedMemoryRatio option. It's computed using the total memory available to the container when running on the Apify platform and a quarter of total system memory when running locally. Max total memory may be overridden by using the APIFY_MEMORY_MBYTES environment variable.

Event loop becomes overloaded if it slows down by more than the maxBlockedMillis option.

CPU tracking is available only on the Apify platform and the CPU overloaded event is read directly off the container and is not configurable.

new Snapshotter([options])

ParamTypeDefault
[options]Object

All Snapshotter parameters are passed via an options object with the following keys:

[options.eventLoopSnapshotIntervalSecs]Number0.5

Defines the interval of measuring the event loop response time.

[options.maxBlockedMillis]Number50

Maximum allowed delay of the event loop in milliseconds. Exceeding this limit overloads the event loop.

[options.memorySnapshotIntervalSecs]Number1

Defines the interval of measuring memory consumption. The measurement itself is resource intensive (25 - 50ms async). Therefore, setting this interval below 1 second is not recommended.

[options.maxUsedMemoryRatio]Number0.7

Defines the maximum ratio of total memory that can be used. Exceeding this limit overloads the memory.

[options.snapshotHistorySecs]Number60

Sets the interval in seconds for which a history of resource snapshots will be kept. Increasing this to very high numbers will affect performance.

snapshotter.start() ⇒ Promise

Starts capturing snapshots at configured intervals.

snapshotter.stop() ⇒ Promise

Stops all resource capturing.

snapshotter.getMemorySample([sampleDurationMillis]) ⇒ Array

Returns a sample of latest memory snapshots, with the size of the sample defined by the sampleDurationMillis parameter. If omitted, it returns a full snapshot history.

ParamType
[sampleDurationMillis]Number
